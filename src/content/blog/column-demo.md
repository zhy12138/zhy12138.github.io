# Demo 说明

这篇文章用于维护 `专栏` tag 的专属能力。只要 frontmatter 中包含 `tags: ["专栏"]`，文章页就会启用左侧目录、阅读进度、滚动高亮和标题编号。

`专栏` 适合一组连续文章，例如“从零开始构建一个推理引擎”。

# 编号规则

`专栏` 从一级标题开始编号。一级标题显示为 `1`、`2`，二级标题显示为 `1.1`、`1.2`，三级标题显示为 `1.1.1`。

目录和正文标题会使用同一套编号，方便长文跳转和引用。

## 二级标题示例

这个标题会显示为 `2.1`。

### 三级标题示例

这个标题会显示为 `2.1.1`。

## 另一个二级标题

这个标题会显示为 `2.2`。

# 内联标记

专栏正文中的 `**加粗**` 会显示为和课程笔记一致的重点标记，例如 **关键结论**。

专栏正文中的 `*斜体*` 会显示为非倾斜的包框标记，例如 *术语提示*。

# 最小写法

新建专栏文章时，frontmatter 至少需要包含 `专栏` tag：

```md
---
title: "从零开始构建一个推理引擎 01"
description: "推理引擎系列第一篇。"
pubDate: 2026-07-21
draft: false
tags: ["专栏", "inference-engine"]
---

# 背景

## 目标

## 非目标

# 架构

## 模块划分
```

# 维护方式

- 如果只是普通分类，不需要目录和编号，不要添加 `专栏` tag。
- 如果文章需要课程笔记的选择题、代码块增强和知识点整理编号规则，使用 `课程笔记` tag。
- 图片等静态资源建议放在 `public/blog/<文章 slug>/` 下，并在正文中用 `/blog/<文章 slug>/文件名` 引用。
